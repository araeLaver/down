## ■ 사업계획서 (10장 이내)

---

### 1. 아이템명

**렌트미(RentMe) — AI 기반 세입자 신뢰 프로필 플랫폼**

세입자가 자신의 신뢰도를 공인 인증으로 증명하고, AI가 집주인 맞춤형 자기소개를 생성하는 프롭테크 플랫폼입니다. 재직·소득·신용 인증과 이전 집주인 레퍼런스를 종합하여 신뢰점수(Trust Score)를 산출하고, 집주인에게 검증된 디지털 프로필을 제시합니다.

---

### 2. 착안 배경

**① 전세사기 이후 무너진 임대차 시장 신뢰**
- 전세사기 피해자 3만 명 돌파(국토부, 2025.7), 피해액 2.5조원 이상. 피해자의 75%가 2030 청년층
- 전세사기 이후 월세 전환 가속화 — 서울 빌라 임대차 중 월세 비중 60%(2025)
- 임대인·임차인 상호 불신 심화 → 거래 지연, 공실 증가, 보증금 분쟁 빈발

**② 세입자의 신뢰 증명 수단 전무**
- 2026년 전세 수요 23.5만 호 vs 입주 물량 8.9만 호 — 14만 호 수급 불균형. 세입자 간 경쟁 심화
- 세입자가 자신의 소득·신용·거주 이력을 체계적으로 증명할 수단이 전무
- 집주인은 세입자 평가 기준이 "느낌"과 "주관적 인상"에 의존 → 정보 비대칭

**③ 기존 플랫폼의 한계 — "매물" 중심, "사람" 부재**
- 직방/다방/네이버 부동산: 매물 정보(가격·사진·위치) 중심 → 세입자 프로필 기능 없음
- 안심전세(전세사기 방지앱): 건물/집주인 위험도 체크 → 세입자 관점의 신뢰 증명 지원 없음
- 부동산 중개 플랫폼: 매물 매칭에 집중 → 세입자-집주인 간 신뢰 구축 미지원

**④ 핵심 착안점**
"매물이 아닌 사람에 집중" — 세입자의 신뢰를 정량화하면, 집주인은 안심하고 세입자를 선택할 수 있고, 세입자는 "좋은 세입자"임을 증명하여 더 좋은 조건의 집을 구할 수 있습니다. 임대차 시장의 정보 비대칭을 기술로 해소하는 것이 렌트미의 출발점입니다.

### 3. 사업 계획

**① 서비스 구조**

렌트미는 세입자와 집주인 양쪽을 연결하는 투사이드(Two-sided) 플랫폼입니다.

- **세입자 측**: 3단계 온보딩(기본정보 → 라이프스타일 → AI 자기소개)으로 5분 이내 프로필 완성 → 공인 인증(재직/소득/신용)으로 신뢰점수 획득 → 프로필 링크를 집주인에게 공유
- **집주인 측**: 전용 대시보드에서 세입자 프로필 검색·필터링·즐겨찾기, 매물 등록(CRUD + 이미지), 1:1 메시징으로 세입자와 직접 소통

**② 3대 핵심 기능**

**(a) 신뢰점수(Trust Score) — 다차원 정량 평가 (0~120점)**

| 평가 항목 | 배점 | 산출 기준 |
|:----------|:----:|:----------|
| 프로필 완성 | 20점 | 기본정보 + 라이프스타일 + 자기소개 입력 완료 |
| 재직 인증 | 25점 | CODEF API → 국민연금 가입이력 조회 확인 |
| 소득 인증 | 25점 | CODEF API → 건강보험료 납부내역 기반 소득 추정 |
| 신용 인증 | 10~20점 | NICE 신용점수 등급별 차등 (1등급 20점 / 2등급 15점 / 3등급 10점) |
| 집주인 레퍼런스 | +30점/건 | 추천 의사 有 + 평균 평점 3.5↑ 시 가점 (비추천 시 -20점 감점) |

등급: Excellent(80+점, 초록) / Good(50~79점, 파랑) / Fair(20~49점, 노랑) / Low(~19점, 회색)

**(b) 공인 인증 시스템 — 서류 없이 원클릭**
- CODEF API: 국민연금 가입이력(재직 확인), 건강보험료(소득 추정), NICE 신용점수(신용등급) 실시간 조회
- NICE 본인인증: CI/DI 발급으로 중복가입 방지 + 본인 확인
- SMS OTP: 6자리 인증번호 전화번호 검증 (NHN Cloud/Twilio 연동)
- RSA 암호화: 주민번호 등 민감정보 전송 시 공개키 암호화 적용

**(c) AI 자기소개 생성 — OpenAI GPT-4o-mini**
- 프로필 데이터(연령대, 가족 구성, 반려동물, 흡연 여부, 체류 패턴, 희망 거주기간)를 분석
- 집주인 관점에서 최적화된 300자 이내 맞춤형 자기소개를 자동 생성
- 정중하고 신뢰감 있는 톤, "안녕하세요"로 시작

### 4. 보유 기술

**① 기술 스택 (전체 직접 구현 완료)**

| 영역 | 기술 |
|:-----|:-----|

| AI | OpenAI GPT-4o-mini (자기소개 생성, temperature 0.7, 300자 제한) |
| 인증 연동 | CODEF(재직/소득/신용), NICE(본인인증 CI/DI), SMS OTP(NHN Cloud/Twilio) |
| 소셜 로그인 | 카카오, 네이버, 구글 OAuth |
| 파일 저장 | AWS S3, Cloudflare R2 (이미지 sharp WebP 최적화) |
| 보안 | Rate Limiting(인증 10회/분, API 60회/분), CSRF 방지, httpOnly/Secure 쿠키, RSA 암호화, XSS 방지 |
| 에러 추적 | Sentry |
| 테스트 | Vitest(단위), Playwright(E2E) |

**② 핵심 기술 차별점**

- **신뢰점수 알고리즘**: 5개 차원(프로필/재직/소득/신용/레퍼런스) 인증 데이터를 가중 합산하여 0~120점 산출. 레퍼런스 긍정(+30)/부정(-20) 가감점 로직으로 허위 프로필 억제. 4단계 등급 시각화
- **3중 공인 인증**: CODEF(국민연금·건강보험·NICE 신용) + NICE 본인인증(CI/DI) + SMS OTP 전화번호 인증. 서류 제출 없이 원클릭 인증 완료. RSA 암호화로 민감정보 보호
- **집주인 레퍼런스 시스템**: 세입자가 이전 집주인 정보 입력 → 시스템이 고유 인증 토큰(7일 유효, 암호학적 보안 토큰) 생성 → 집주인은 로그인 없이 토큰으로 설문 접근 → 4항목(월세 납부, 시설 관리, 이웃 관계, 퇴거 상태) 1~5점 평가 + 추천 여부 + 코멘트 → 결과 자동 신뢰점수 반영
- **AI 자기소개 엔진**: OpenAI GPT-4o-mini 기반 프로필 분석 → 집주인 니즈에 맞춘 맞춤형 소개 자동 생성. API 미설정 시 템플릿 기반 폴백 제공

**③ MVP 완성도 — 99% (프로덕션 레디)**
