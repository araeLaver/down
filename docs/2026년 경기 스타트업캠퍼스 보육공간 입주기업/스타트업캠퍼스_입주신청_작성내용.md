## [지정양식] 사업계획서 작성 내용

### I. 기업 현황 및 입주공간 필요성

#### 1. 창업 배경 및 동기

전세사기 피해자 3만 명, 피해액 2조 5천억 원. 2024년 이후 한국 임대차 시장의 신뢰는 완전히 무너졌습니다.

그런데 **문제는 양방향**입니다. 세입자는 사기 집주인을 두려워하고, 집주인은 월세를 안 낼 세입자를 두려워합니다. 현재 세입자가 자신의 신뢰를 증명할 수 있는 플랫폼은 **제로**입니다.

직방, 다방, 네이버부동산 — 전부 "매물" 검색 플랫폼이지, **"사람"을 다루는 플랫폼은 없습니다.**

10년간 풀스택 개발자로 **금융 스크래핑(헥토데이터 3년)**, **다중인증(IBK)**, **매칭 알고리즘(에너지마켓플레이스)**을 직접 개발한 경험을 바탕으로, 세입자와 집주인 간의 신뢰를 숫자로 만드는 플랫폼 입주해를 개발했습니다.

**MVP 자체 개발 완료, 외주비 0원.**

#### 2. 제품/서비스 개요

**서비스명**: 입주해 (입주해) — "세입자의 링크드인"

**3대 핵심 기능:**

| 기능 | 설명 | 리소스 비중 |
|------|------|:----------:|
| **신뢰점수 (Trust Score)** | 0~120점. 프로필 완성도, 재직 증명, 소득 수준, 신용 등급, 임대인 레퍼런스 5차원 종합 평가 | 40% |
| **자동 인증 (자체 스크래핑)** | 홈택스/건강보험 등 자체 스크래핑으로 재직·소득 원클릭 검증. 서류 제출 없음. 원본 데이터 미저장 | 30% |
| **시맨틱 매칭 (OpenSearch)** | 벡터 검색으로 세입자-매물 자동 매칭. "역삼역 10분, 월세 80만 이하" 자연어 검색 | 30% |

**추가 기능:**
- AI 자기소개서 생성 (OpenAI GPT-4o-mini)
- 이전 임대인 레퍼런스 (SMS 설문)
- 임대인용 세입자 프로필 열람 (무료)

**기술 스택:**

- Backend: Next.js App Router, PostgreSQL, JWT
- 검색 엔진: OpenSearch 2.x (k-NN 벡터 검색)
- AI: OpenAI GPT-4o-mini (자기소개서 생성)
- 인증: 자체 스크래핑 (홈택스, 건강보험 등), NICE 본인인증, SMS OTP
- 보안: RSA 암호화, Rate Limiting, CSRF 방지
- 인프라: Vercel, AWS S3, Sentry, Vitest/Playwright

#### 4. 입주 필요성 (왜 판교 스타트업캠퍼스인가)

**① 프롭테크 생태계 중심지**
- 직방, 다방, 호갱노노, 집토스 등 **프롭테크 기업이 판교에 집중**
- B2B 제휴(중개업소 SaaS), 데이터 파트너십 네트워킹에 최적
- 부동산 관련 VC/AC가 판교-강남에 밀집

**② 사업장 주소 확보**
- 예비창업자로 사업자등록을 위한 사업장 주소가 필요
- 입주 확정 시 법인 설립 + 사업자등록을 판교 주소로 진행
- "판교 소재 프롭테크 스타트업" → B2B 영업 시 신뢰도 상승

**③ 단계별 성장 지원**
- 예비(3층) → 초기(5층) → 성장(8층)까지 연속 보육
- 교육장, 멘토룸, 회의실 등 인프라 활용으로 초기 비용 최소화
- 연 4회 네트워킹 행사 → 투자자/멘토 접근

**④ 타겟 시장 접근성**
- 입주해 1차 타겟: 수도권 25~35세 직장인 세입자
- 판교/강남/분당은 임대차 수요 최상위 지역 → 베타 테스트에 최적
- 수도권 중개업소 B2B 영업 접근성

**⑤ 전업 준비 완료**
- 정부 지원사업 선정 시 퇴사 → 판교 상주 전업
- 겸업동의서 준비 완료, 전업 전환 조건 명확
- 경기도 광주시 거주 → 판교까지 50분 이내

#### 2. 기술성 (30점)

**핵심 기술 4가지:**

**① 자체 스크래핑 인증 시스템**
- 홈택스, 건강보험 등에서 재직/소득 정보를 **자체 스크래핑으로 자동 검증**
- 외부 유료 API 의존 없이 자체 모듈로 운영 → 비용 최소화
- **원본 데이터 미저장** — 점수 산출 후 즉시 삭제 (개인정보 최소 수집 원칙)
- 경력: **헥토데이터(구 쿠콘)에서 3년간 금융 스크래핑 솔루션 개발** — 이 분야 최고 수준의 실무 경험

**② OpenSearch 시맨틱 검색/매칭**
- k-NN 벡터 검색으로 세입자 프로필과 매물을 자동 매칭
- 자연어 검색: "역삼역 10분, 월세 80만 이하, 반려동물 가능" → 벡터로 변환 → 유사도 매칭
- 시장 인사이트 대시보드: 지역별 평균 신뢰점수, 경쟁률 등 데이터 시각화

**③ AI 자기소개서 생성 (OpenAI)**
- 사용자 입력 정보(직업, 생활패턴, 성격)를 바탕으로 GPT-4o-mini가 맞춤형 자기소개서 자동 생성
- 집주인이 "이 사람은 어떤 사람인가"를 3초 만에 파악 가능
- 템플릿 기반이 아닌 **개인화된 자연어 생성**

**④ 신뢰점수(Trust Score) 산출 알고리즘**
- 5차원 평가: 프로필(20) + 재직(25) + 소득(25) + 신용(25) + 레퍼런스(25) = 120점 만점
- 각 차원별 가중치 동적 조정 (데이터 축적에 따라 ML로 최적화)
- 신용조회업에 해당하지 않도록 **자체 산출 점수**로 설계 (법적 리스크 회피)

**기술 차별점:**
- 직방/다방/네이버부동산 = "매물 DB" / 입주해 = **"사람 DB"** → 데이터 성격 자체가 다름
- 세입자 프로필 + 공인 인증 + 집주인 레퍼런스 + 시맨틱 매칭을 결합한 서비스 **국내외 부재**
- MVP 자체 개발 완료 — 기술 검증 완료 상태

**왜 직방/다방이 따라하기 어려운가:**
- 직방/다방의 수익모델 = 중개업소 광고비. 세입자에게 돈을 받는 건 기존 고객(중개업소)과 이해충돌.
- 입주해는 "사람 DB"로 데이터 네트워크 효과 → 프로필 축적 → 매칭 정확도 상승 → 후발주자 진입장벽.
- 오히려 직방/다방과 API 연동이 양쪽 윈윈.
