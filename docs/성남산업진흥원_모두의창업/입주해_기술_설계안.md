# 🗄️ 입주해(Ipjuhae) Trust Score DB 설계 및 산출 로직 (초안)

## 1. 데이터베이스 테이블 설계 (PostgreSQL)

### **Table: users (사용자 기본 정보)**
| Column | Type | Description |
| :--- | :--- | :--- |
| id | UUID | 기본 키 (PK) |
| email | String | 계정 이메일 (Unique) |
| user_type | Enum | 'TENANT', 'LANDLORD' |
| trust_score | Integer | 최종 산출된 신뢰 점수 (0~100) |
| trust_grade | String | 등급 (Platinum, Gold, Silver, Bronze) |

### **Table: scrap_data (스크래핑 원본 데이터 - 암호화 저장)**
| Column | Type | Description |
| :--- | :--- | :--- |
| user_id | UUID | 사용자 외래 키 (FK) |
| income_annual | BigInt | 연간 소득 금액 (홈택스) |
| job_type | String | 고용 형태 (정규직, 계약직, 프리랜서 등) |
| employment_months| Integer | 재직 기간 (월 수) |
| credit_rating | Integer | 신용 점수 (NICE/KCB) |
| last_scraped_at | Timestamp| 마지막 데이터 갱신 시각 |

### **Table: references (임대인 평판 데이터)**
| Column | Type | Description |
| :--- | :--- | :--- |
| tenant_id | UUID | 세입자 ID (FK) |
| landlord_id | UUID | 평판을 작성한 임대인 ID (FK) |
| rent_punctuality | Integer | 월세 납부 성실도 (1~5) |
| facility_care | Integer | 시설 관리 상태 (1~5) |
| comment | Text | 정성적 평가 의견 |

## 2. Trust Score 산출 로직 (Algorithm Logic)

```javascript
/**
 * Trust Score 산출 함수 (초안)
 */
function calculateTrustScore(data) {
    let score = 0;

    // 1. 경제적 신용 (40%)
    // - 연소득 3천만원 이상 20점, 재직 1년 이상 20점 가산
    const incomeScore = data.income_annual >= 30000000 ? 20 : (data.income_annual / 30000000) * 20;
    const employmentScore = data.employment_months >= 12 ? 20 : (data.employment_months / 12) * 20;
    score += (incomeScore + employmentScore);

    // 2. 금융 신뢰도 (30%)
    // - 신용점수 900점 이상 30점, 700점 이하 10점 등 비례 배분
    const creditScore = (data.credit_rating / 1000) * 30;
    score += creditScore;

    // 3. 주거 평판 (20%) - 이전 임대인 평판 평균값
    if (data.reference_avg) {
        score += (data.reference_avg / 5) * 20;
    } else {
        score += 10; // 평판 없을 시 기본 점수 부여
    }

    // 4. 활동 및 본인인증 (10%)
    // - 본인인증 완료 시 10점 가산
    if (data.is_authenticated) {
        score += 10;
    }

    return Math.round(score);
}
```
