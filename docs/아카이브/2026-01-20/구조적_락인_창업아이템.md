### 1. 법적 의무 (안 하면 벌금/과태료)

| 의무 | 대상 | 벌칙 |
|------|------|------|
| **전자세금계산서** | 법인 전체, 개인 8천만↑ | 가산세 |
| **4대보험 신고** | 직원 있는 모든 사업장 | 과태료 |

| **개인정보 관리** | 개인정보 취급 사업자 | 과태료 |
| **ESG 공시** | 상장사 (2025~) | 상장 요건 |

### 2. 데이터 락인 (쌓이면 이탈 불가)

```

│  • 회계 데이터: 3년치 쌓이면 이동 고통                          │
```

## 아이템 1: 배달앱 통합 관리 + 자체 주문 시스템

### Pain Point

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  현실:                                                           │
│  • 매출 48.8%가 배달앱 의존                                      │
│  • 수수료 16~29% (인건비보다 높음)                               │
│  • 배민/쿠팡/요기요 각각 관리 = 시간 낭비                        │
│  • 주문 놓치면 = 별점 테러                                       │
│                                                                 │
│  사장님 바람:                                                    │
│  • 배달앱 수수료 줄이고 싶다                                     │
│  • 자체 주문 받고 싶다 (단골 고객)                               │
│  • 한 화면에서 다 보고 싶다                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 솔루션

```
서비스명: 오더플러스 / 사장님오더

핵심 기능:
1. 배달앱 통합 대시보드 (배민+쿠팡+요기요 한 화면)
2. 자체 주문 페이지 (카카오톡 연동)
3. 단골 고객 관리 (재주문 유도)
4. 주문 → 주방 프린터 자동 출력
5. 매출/수수료 분석 리포트
```

### 왜 락인되는가?

```
[데이터 락인]
• 단골 고객 목록 축적
• 주문 히스토리 (메뉴별 인기도)
• 매출 데이터 분석

[전환 비용]
• 프린터/POS 연동 재설정
• 고객에게 새 주문 링크 안내
• 직원 재교육

[네트워크 효과]
• 자체 주문 고객 늘어날수록 가치 증가
```

### 수익 모델

| 플랜 | 가격 | 기능 |
|------|------|------|
| Basic | ₩29,000/월 | 통합 대시보드 |
| Pro | ₩49,000/월 | 자체 주문 + 단골 관리 |
| Premium | ₩79,000/월 | 마케팅 자동화 |

### 시장 규모

```
배달앱 이용 자영업자: 약 50만 개
월 ₩49,000 × 50만 = ₩245억/월 (잠재 시장)
점유율 1%만 해도 = ₩2.45억/월
```

## 아이템 3: 전자세금계산서 + 매입/매출 관리

### Pain Point

```
법적 의무:
• 법인: 전자세금계산서 의무 (2011~)
• 개인: 연매출 8천만 이상 의무 (2024~)
• 미발행 시 가산세: 공급가액의 1~2%

현실:
• 홈택스 불편 → 별도 솔루션 필요
• 세금계산서 + 매입/매출 따로 관리
• 부가세 신고 때마다 야근
```

### 솔루션

```
서비스명: 세금플러스 / 간편세금

핵심 기능:
1. 전자세금계산서 발행 (홈택스 연동)
2. 매입/매출 자동 대사
3. 부가세 신고서 자동 생성
4. 거래처 관리
5. 미수금/미지급 알림
```

### 왜 락인되는가?

```
[법적 의무]
• 전자세금계산서 = 안 하면 가산세
• 부가세 신고 = 3년치 데이터 필요

[데이터 락인]
• 거래처 DB 축적
• 3년 매입/매출 히스토리
• 세무사 연동 (데이터 공유)

[전환 비용]
• 기존 데이터 마이그레이션
• 거래처 정보 재입력
• 세무사 재연동
```

### 경쟁 분석

| 서비스 | 가격 | 특징 |
|--------|------|------|
| 홈택스 | 무료 | 불편, 기능 제한 |
| 더존 | 비쌈 | 대기업용 |
| 자비스 | 중간 | 법인용 |
| **틈새** | - | **소규모 사업자 특화** |

## 아이템 4: 직원 근태/급여 관리 (5인 이상 필수)

### 법적 배경

```
근로기준법:
• 5인 이상 사업장 = 근로시간 기록 의무
• 52시간 근무제 위반 시 = 과태료 + 형사처벌
• 연장근로 수당 미지급 = 임금체불

현실:
• 엑셀 관리 = 증거 능력 없음
• 수기 출퇴근부 = 조작 의심
• 급여 계산 실수 = 노동청 신고
```

### 솔루션

```
서비스명: 출퇴근플러스 / 근태메이트

핵심 기능:
1. 모바일 출퇴근 체크 (GPS/QR)
2. 근로시간 자동 계산 (연장/야간/휴일)
3. 급여 자동 계산
4. 4대보험 신고 연동
5. 근로계약서 전자서명
6. 연차 관리
```

### 왜 락인되는가?

```
[법적 의무]
• 근로시간 기록 3년 보관 의무
• 급여 명세서 보관 의무

[데이터 락인]
• 직원 히스토리 (입사~퇴사)
• 급여 데이터 (연말정산, 4대보험)
• 연차 이월 데이터

[전환 비용]
• 급여일에 시스템 교체 = 재앙
• 직원 앱 재설치 + 재교육
• 세무사/노무사 연동 재설정
```

### 시장 규모

```
5인 이상 사업장: 약 60만 개
- 기존 HR 솔루션 사용: 20% (플렉스, 시프티)
- 엑셀/수기 관리: 80% (48만 개)

월 ₩30,000 × 48만 = ₩144억/월 (잠재 시장)
```

## 아이템 5: 재고 관리 + 발주 자동화 (요식업/소매업)

```

• 재고 과잉 = 폐기 = 비용 손실
```

```
서비스명: 재고플러스 / 발주메이트

핵심 기능:

3. 자동 발주 알림 (재고 부족 시)
```

### 왜 락인되는가?

```

│  • 자체 주문 + 통합 관리 = 차별화                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 인사이트

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│         "쓸 수밖에 없는 구조" 만드는 3가지 조건                  │
│                                                                 │
│  1. 법적 의무 연동                                               │
│     → 안 쓰면 벌금/과태료                                        │
│     → 예: 근로시간 기록, 전자세금계산서                          │
│                                                                 │
│  2. 데이터 축적 = 자산                                           │
│     → 3년 쌓이면 이동 불가                                       │
│     → 예: 급여 데이터, 고객 히스토리, 재고 패턴                  │
│                                                                 │
│  3. 업무 프로세스 통합                                           │
│     → 여러 시스템 대체 = 교체 비용 높음                          │
│     → 예: 출퇴근 + 급여 + 4대보험 + 연말정산                     │
│                                                                 │
│  이 3가지 중 2개 이상 충족하면 = 강력한 락인                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
