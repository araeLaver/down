## 1. 문제 인식 (Problem) — 창업 아이템의 필요성

### (1) 국내외 시장 현황 및 문제점

**전세에서 월세로의 전환이 가속되면서, 임대차 시장의 '신뢰 인프라'가 부재합니다.**

| 지표 | 수치 | 출처 |
|:-----|:-----|:-----|
| 1인 가구 수 | 804만 (전체 가구 36.1%) | 통계청, 2024 |
| 월세 비중 | 60.2% (역대 최고) | 국토교통부, 2025 |
| 서울 원룸 전세 평균 | 2억 1,700만원 | KB부동산, 2024 |
| 월세 보증금 평균 | 500~1,000만원 | 부동산114, 2024 |
| 전세사기 공식 피해자 | 2.5만명+ | 정부 발표, 2024 |
| 청년 무주택 비율 | 73.2% | 통계청, 2023 |

**세입자와 집주인 양쪽 모두 해결되지 않는 문제를 안고 있습니다.**

| 이해관계자 | 핵심 문제 | 현재 상황 |
|:-----------|:---------|:---------|
| **세입자** | "나는 좋은 세입자인데 증명할 방법이 없다" | 이전 임대에서 문제 없이 살았던 기록 활용 불가. 안정적 직장·소득이 있어도 집주인이 확인할 방법 없음. 결국 '보증금'으로만 신뢰 담보 |
| **집주인** | "이 사람이 월세를 잘 낼지 어떻게 아나" | 세입자 사전 확인 수단 = 신분증, 재직증명서 정도. 월세 연체·시설 파손·민원 이력 확인 불가. 전세사기 이후 임대인도 신중해졌으나 검증 수단 없음 |

### (2) 기존 서비스의 한계

| 기존 서비스 | 포지션 | 한계 | 렌트미 차별점 |
|:-----------|:-------|:-----|:-------------|
| 직방/다방 | 매물 검색 | 세입자 프로필·인증 기능 전무 | 세입자 중심 신뢰 프로필 |
| 피터팬의 좋은방 구하기 | 직거래 커뮤니티 | 신뢰 보장 없음, 허위매물 리스크 | API 자동 인증, 위변조 불가 |
| 네이버 부동산 | 정보 제공 | 매칭 기능 없음 | 세입자-집주인 조건 매칭 |
| **해외 사례** | | | |
| Zillow (미국) | 매물 중심 | 세입자 인증 제한적 | 다층 인증 + 집주인 레퍼런스 |
| OpenRent (영국) | 세입자 레퍼런스 존재 | 한국 시장 특성(전세/월세) 미반영 | 한국형 마이데이터 인증 체계 |

**핵심**: 한국에는 세입자 관점의 신뢰 인증 플랫폼이 전무합니다. 직방·다방은 '매물' 중심이며, 렌트미는 '사람(세입자)' 중심 — 경쟁이 아닌 보완 관계입니다.

## 2. 실현 가능성 (Solution) — 창업 아이템의 개발 계획

### (1) 핵심 솔루션 — "세입자의 LinkedIn"

렌트미는 세입자의 **신뢰 프로필**을 생성하여, 신뢰를 '자산'으로 만드는 플랫폼입니다.

| 핵심 기능 | 설명 | 기술 기반 | 차별화 |
|:---------|:-----|:---------|:-------|
| **마이데이터 연계 자동 인증** | 재직·소득·신용정보를 공공/금융 데이터 API로 실시간 검증 | 마이데이터 API 연동 (건강보험공단·국세청·CB사) | 자동화, 위변조 불가, 1분 내 완료 |
| **집주인 레퍼런스** | 이전 집주인에게 고유코드 전달 → 간편 평가 | 고유코드 기반 비대면 평가 시스템 | **한국 최초**. 월세 연체 여부, 시설 관리 상태 증명 |
| **LLM 기반 AI 자기소개서** | 프로필·라이프스타일 데이터 기반 맞춤 자기소개 자동 생성 | 거대언어모델(LLM) 자연어 생성 | 집주인 관점 최적화, 150자 내외 |
| **다층 신뢰 스코어링** | 인증 + 레퍼런스 + 프로필을 종합 분석하여 0~100점 산출 | 가중 평균 스코어링 알고리즘 | 높을수록 보증금 협상력 상승 |
| **세입자-집주인 매칭** | 신뢰점수 기반 조건 매칭, 집주인 맞춤 세입자 추천 | 가중 필터링 + 추천 알고리즘 | 집주인이 원하는 조건의 세입자를 자동 추천 |

**신뢰점수 산출 방식:**
```
기본 20점 + 재직인증(25) + 소득인증(25) + 신용인증(20) + 집주인 레퍼런스(30) = 최대 120점 → 100점 환산
예시) 20 + 25(재직) + 25(소득) + 17(신용3등급) = 87점
```

### (3) 협약기간 내 개발 계획 (8개월)

**1단계 (M1~M4, 2026.6~9)** — MVP 고도화 + 시장 검증

- M1(6월): 마이데이터 연계 재직·소득 자동 인증 연동, 프로필 생성 플로우 완성, 법인 설립
- M2(7월): LLM 자기소개서 정확도 개선, 집주인용 세입자 탐색 페이지, UI/UX 디자인 외주
- M3(8월): 셰어하우스 2곳 B2B 제휴, 베타 테스트 200명 모집
- M4(9월): 베타 피드백 반영, 서비스 정식 오픈, 마케터 채용

**2단계 (M5~M8, 2026.10~2027.1)** — 사업화·성장

- M5(10월): CB사 신용 인증 API 연동, 다층 신뢰 스코어링 시스템 런칭, SNS 마케팅 시작
- M6(11월): 집주인 구독 결제 시스템, 대학가 타겟 마케팅
- M7(12월): 집주인 매칭 알림 시스템, 부동산 인플루언서 협업
- M8(1월): 성과 정리, IR 자료 제작, Seed 투자유치 준비

### (4) 경쟁력 확보 방안

- **데이터 해자**: 집주인 레퍼런스 DB는 시간이 지날수록 축적 → 후발 주자 진입 어려움
- **비즈니스 모델 충돌**: 직방/다방은 중개사 광고 수익(70~80%) 의존 → 세입자 인증 서비스로 전환 시 기존 모델과 충돌
- **네트워크 효과**: 세입자↔집주인 양면 시장 — 사용자가 많을수록 매칭 가치 상승
- **최소 비용 운영**: 클라우드 서버리스 아키텍처로 월 10만원 이하 운영 가능, 1인 개발·운영 체제

## 3. 성장전략 (Scale-up) — 사업화 추진 전략

### (1) 시장 규모

| 구분 | 규모 | 산출 근거 |
|:-----|-----:|:----------|
| TAM | 약 2조원 | 한국 임차 가구 900만, 연 200만 건 임차 거래 |
| SAM | 약 6,000억원 | 1인 가구 임차 550만, 월세 거래 120만 건/년 |
| SOM | 약 30억원 | 수도권 1인 월세 50만 건 × 점유율 5% (3년 목표) |

- 월세 비중 60.2% → 역대 최고, 전세→월세 전환 가속 중
- 무보증 월세 서울 연 200건+ 증가 추세 (헤럴드경제)
- 전세사기 이후 신뢰 기반 거래 수요 폭발적 증가

### (2) 수익 모델 (BM)

```
[세입자 - B2C]
 · 기본 (무료): 프로필 생성, 기본 인증, AI 자기소개 1회
 · 프리미엄: ₩9,900/월 (우선 노출, 무제한 AI 자기소개, 레퍼런스 무제한)

[집주인 - B2B]
 · 프로필 열람: ₩3,900/건 (검증된 세입자 연락처 + 상세 프로필)
 · 월 구독: ₩29,900/월 (무제한 열람, 매칭 알림, 우선 지원)

```

| 수익원 | 대상 | 가격 | 예상 비중 |
|:-------|:-----|:-----|:---------:|
| 프리미엄 프로필 | 세입자 | ₩9,900/월 | 40% |
| 프로필 열람 | 집주인 | ₩3,900/건 | 30% |
| 집주인 구독 | 집주인 | ₩29,900/월 | 30% |

**Unit Economics:**
- 세입자 CAC: ₩5,000 (SNS 마케팅) / LTV: ₩59,400 (9,900 × 6개월) → **LTV:CAC = 12:1**
- 집주인 CAC: ₩20,000 (타겟 광고) / LTV: ₩179,400 (29,900 × 6개월) → **LTV:CAC = 9:1**
- **인프라 자체 충당**: 유료 구독 89명 (월 인프라비 약 89만원 기준)
- **사업 전체 BEP**: 유료 구독 약 500명 + 집주인 열람 300건/월 (인건비·마케팅 포함, 2027년 하반기 예상)
